# Implementation Plan: Simple Toolkit Styles

**Branch**: `feature/simple-toolkit-styles` | **Date**: 2026-02-16 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/simple-toolkit-styles/spec.md`

## Summary

Create a new companion library `Uno.Toolkit.WinUI.Simple` inside the
`uno.toolkit.ui` repository that provides ~45 consumer-facing SDS-themed styles
for 6 Toolkit control families (Card, CardContentControl, Chip, ChipGroup,
Divider, NavigationBar, TabBar/TabBarItem). The implementation mirrors
`Uno.Toolkit.Material` exactly in project structure, entry point class, and
XamlMerge configuration, substituting Material design tokens with SDS design
tokens.

## Dependencies & Execution Order

### Phase Dependencies

- **Phase 1 (Setup)**: No dependencies — can start immediately
- **Phase 2 (US1 — Cards)**: Depends on Phase 1 completion, first checkpoint
- **Phase 3 (US2 — Chips)**: Depends on Phase 1 only (parallel with US1)
- **Phase 4 (US3 — TabBar)**: Depends on Phase 1 only (parallel with US1/US2)
- **Phase 5 (US4 — Divider)**: Depends on Phase 1 only (parallel)
- **Phase 6 (US5 — NavBar)**: Depends on Phase 1 only (parallel)
- **Phase 7 (Polish)**: Depends on all user stories

### Critical Path

Setup → Card (first control validates entire pipeline) → remaining controls in
parallel → `_Common.xaml` aliases → samples → build validation

### Implementation Strategy

**MVP First**: Complete Setup + US1 (Cards) → validate build → then add
remaining controls. Cards are the most commonly used Toolkit control and
validating the build pipeline early prevents rework.

**Incremental Delivery**: Each user story adds one XAML file + updates
`_Common.xaml`. No cross-story dependencies.

## Complexity Tracking

No constitution violations to justify. This is a straightforward replication of
an existing pattern (Material Toolkit) with different design tokens.
